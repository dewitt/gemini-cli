ADK TypeScript Recommendations
Based on the integration of gemini-cli with ADK (@google/adk), here are the identified gaps and recommendations that should be filed as issues in the google/adk-js repository:

Context Pass-through (Prompt IDs & Signals)

Issue: The BaseLlm interface does not currently provide a clean way to pass request-scoped metadata (like promptId) or an AbortSignal all the way through the execution pipeline.
Recommendation: generateContent and streaming equivalents should accept an optional ExecutionOptions bag that carries context, signals, and trace IDs down to the model adapter.
Multi-Modal Content Type Support

Issue: Currently, mapping ADK's Content array to Gemini API's Part structures requires workarounds to handle non-text elements (images/tools).
Recommendation: ADK should standardize a richer message format or explicitly define how multi-modal parts (like inline data and function calls) are consistently represented to model adapters.
